# VayLord 에이전트 경제 — 기준선 (v0.1)

> lqez/vaylord-workspace#52. OpenMMO ECONOMY.md 연구에서 채택한 착취-방지
> 불변식을 VayLord(자율 에이전트 MMO)에 맞게 고정한 **설계 기준선**입니다.
> 게임 서버 착수 전의 문서 — 불변식만 짧게 동결하고, 균형 상수는 추후 시뮬로 잡습니다.

## 1. 불변식 (어기면 경제가 무너지는 것들)

1. **LLM은 가격을 정하지 않는다.** 서버가 품목별 `[최저, 최고]` 가격 밴드를
   계산하고, 에이전트(LLM)는 밴드 **안의 수정자만** 선택한다.
2. **어떤 협상 조합으로도 무한 차익 루프가 불가능해야 한다.**
   상시 성립: `상점의 최저 매입가 > 상점의 최고 매도가` (스프레드 보장).
3. **캐릭터/에이전트 생성은 환금 가능 자산을 발행하지 않는다.**
   시작 자금 0, 시작 장비는 `basePrice` 없는 비매품.
4. **모든 반복 행동에는 쿨다운 또는 체감 수익이 있다.** 협상 쿨다운,
   품목별 일일 예산, 동일 상대 반복 거래 감쇠.
5. **싱크가 소스를 따라잡는다.** 수리·감정·통행세·시장세 = 골드 제거 장치.
   신규 유입(전리품·채굴)과 제거의 비율을 대시보드로 상시 계측.

## 2. VayLord 매핑

- **통화**: 단일 통화(서버 64bit 정수), 표시만 금/은/동. 가문별 통화는 만들지
  않는다(교환비 착취 표면이 됨) — 가문 정체성은 **가격 밴드 폭**으로:
  스팀메시 상권은 밴드 좁게(정찰제), 무법지대는 넓게(흥정).
- **공급 지도 = world-creator 산출물**: 자원 레이어(마나석 등)·POI
  (lqez/vaylord-workspace#51)·세력권(lqez/vaylord-workspace#34)의 **지역 편재**가
  교역 동기다. 자원 export에 재생 속도/채굴 난도 메타 추가 검토
  (lqez/vaylord-workspace#32 연계).
- **에이전트↔에이전트 거래는 차단할 수 없다**(에이전트가 본체인 게임).
  OpenMMO의 "NPC↔NPC 차단" 대신: 거래 쿨다운 + 위시리스트 기반 수요(원하는
  것만 산다) + **거래세 싱크**를 1급 규칙으로.
- **비상인 에이전트**: 유한 지갑 + 주기 수입(직업 시뮬) + 좁은 위시리스트.
  상인 에이전트: 넓은 카탈로그 + 좁은 밴드 = 가격 앵커.
- **미감정 아이템**: 종류는 보이되 개별 어픽스는 감정 전 비공개(위키 스포일러
  방지) — 감정비는 싱크.

## 3. 에이전트 행동 API와의 접점

`trade` MCP 툴(lqez/vaylord-workspace#51의 인터페이스 초안에 포함)은 다음 계약을
전제로 한다:

```
trade(target, offer[], want[], modifier)  // modifier ∈ 서버가 준 밴드
→ 서버: 밴드 검증 → 불변식 2 확인 → 쿨다운/예산 차감 → 체결/거절
```

LLM 프롬프트에는 잔액·밴드·쿨다운 상태를 구조화로 제공하고, 결정만 받는다.

## 4. 미결(owner 결정 필요)

- 가문 간 관세/통행세율 — 세계관(세력 관계)과 함께.
- 에이전트 "급여"의 원천 서사(패트론 지원금? 가문 배급?).
- 시즌 리셋/인플레 억제 정책.
